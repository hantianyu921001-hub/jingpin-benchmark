# 2026-07-03

## 新增：图片 + 网页抓取数据录入功能

### 新增文件
- **extractor.py** (14KB) — 视觉提取引擎，调用 OpenAI 兼容的视觉模型 API（GPT-4V / 通义千问 VL / DeepSeek V4），将参数页截图解析为结构化 JSON，自动映射到 DIMENSION_DEFS 的 11 个维度。支持多图片合并提取。
- **scraper.py** (9.5KB) — 网页截图抓取模块，用 Playwright 加载汽车参数页 URL，支持 5 个站点配置（汽车之家、懂车帝、鸿蒙智行、理想官网、通用），截取参数区域后传给 extractor。
- **.env.example** — 环境变量模板，列出 4 种视觉模型配置方案。

### 修改文件
- **app.py**：新增 4 个路由
  - `POST /api/extract/image` — 上传图片提取数据
  - `POST /api/extract/url` — 输入 URL 抓取并提取
  - `POST /api/extract/save` — 确认后写入 SQLite field_overrides
  - `GET /extract` — 数据录入 HTML 页面（含上传区、URL 输入、预览编辑器、确认保存）
  - 首页导航栏新增「数据录入」入口链接
  - 新增 UPLOAD_DIR 上传目录管理

### 技术要点
- 视觉模型方案：分离式——截图 → 视觉模型提取文字 → 结构化 JSON
- 支持 4 种 API：OpenAI GPT-4V / 通义千问 VL / DeepSeek V4 / 通用兼容 API
- 自动从 .env 文件加载环境变量
- 提取结果在网页端可逐字段编辑后确认写入
- 数据流向：截图/URL → extractor → 预览编辑 → SQLite field_overrides → ETL 刷新
- 全部导入测试通过（managed Python 3.13 + Flask 3.1.3）
- 端到端 API 测试通过（12 个路由全部正常）

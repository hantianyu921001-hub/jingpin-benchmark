## 触发场景

用户提供工信部公告批次链接或文件，要求将其中的新车型补录到竞品表。

## 竞品筛选规则

只录入两类车型，其余跳过：
1. **鸿蒙智行生态新品**：问界、智界、享界、尊界
2. **直接竞品**：与理想现有产品在价格段/车型定位上存在直接竞争关系的车型

## 表操作顺序

### 1. 00_车型（`tbl92hQB5ngDEvRH`）

每个筛选出的车型创建一条记录，关键字段：

```json
{
  "车型统一ID": "<品牌>_<车系>_<年款>",
  "车型排序码": "<4位排序码，见下方品牌映射>",
  "年款/代际": "<年款>",
  "品牌": "<品牌>",
  "车系": "<品牌> <车系>",
  "类型": "<车型类别>",
  "上市状态": "公示期",
  "发布时间": <Unix毫秒时间戳，取工信部批次发布日期，CST=UTC+8>
}
```

### 2. 10_版本价格（`tblWDFfm3M3uX9qx`）

每个工信部产品型号（product code）创建一条记录：

```json
{
  "版本统一ID": "<品牌>_<车系>_<年款>_<工信部产品型号>",
  "车型排序码": "<4位排序码>",
  "能源": "<纯电|增程|插混>",
  "座位数": "<N座>",
  "核心配置摘要": "工信部申报型号:<产品型号>；能源:<能源完整名称>；轮胎:<轮胎规格>",
  "关联车型": [{"id": "<00_车型 record_id>"}]
}
```

### 3. 21_尺寸（`tblROhrR1M84SZrp`）

按"相同 排序码 + 长 + 宽 + 高 + 轴距"分组，每组一条记录：

```json
{
  "配置组": "<全系 (工信部) | 按差异命名，如"低车顶版"、"长轴版">",
  "版本范围": "<产品型号1；产品型号2；...>",
  "车型排序码": "<4位排序码>",
  "长(mm)": <int>,
  "宽(mm)": <int>,
  "高(mm)": <int>,
  "轴距(mm)": <int>,
  "轮胎规格": "<合并去重，多个用；分隔>",
  "关联车型": [{"id": "<00_车型 record_id>"}],
  "适用版本": [{"id": "<10_版本价格 record_id>"}, ...]
}
```

**分组命名原则（多组时）：**
- 车高差异明显 → "低车顶版" / "标准版"
- 车长差异 → "标准版" / "长轴版"
- 单一分组 → "全系 (工信部)"

## 日期时间戳计算

日期时间戳为毫秒级 Unix，CST = UTC+8：

```python
from datetime import datetime, timezone, timedelta
CST = timezone(timedelta(hours=8))
dt = datetime(2026, 6, 10, 0, 0, tzinfo=CST)
ts = int(dt.timestamp() * 1000)  # 1781020800000
```

## 注意事项

- 工信部申报产品型号有时一个车系对应多个产品型号（不同配置/批次），每个型号在 `10_版本价格` 中各建一条记录。
- `21_尺寸` 的 `适用版本` 和 `关联车型` 都是关联字段，值格式为 `[{"id": "recXXXX"}]`。
- 分页读取 `10_版本价格` 时 `--limit` 最大 200，需要用 `offset` 分页。
- 批量写入时每条记录之间 sleep 0.5s，避免并发写冲突（错误码 1254291）。

## 公示期车型特殊规则

**版本等级**：公示期（工信部批次）车型在 `10_版本价格` 中**不填写版本等级**，保持该字段为空。待车型正式上市、官方发布配置版本名称后，再补充版本等级。

原因：工信部公示期仅提供产品型号（如 `BJ6496VALBEV1`），无官方市场营销版本名称（如 "Max"、"Ultra" 等），若强行填写会导致后续正式发布时需大量修改，增加维护成本。
